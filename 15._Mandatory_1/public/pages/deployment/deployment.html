Deployment with Vercel

One way to deploy a node.js application is using Vercel and with this it's easy to setup.

The first thing we have to do is create a repository with our project on Github. If you are completely new to Vercel, the guide here will start from the beginning.

The vercel.json file.

The first thing we need to do after creating the Github repository is to create a vercel.json file. We will insert what is shown in the image below and push it to Github. Remeber if you have several projects in your repository and only want to deploy one of them you have to insert the vercel file in that folder (where you app.js is).

The source where we got it from is here How to deploy an Express API to Vercel - 4. Add vercel.json configuration.

The vercel.json file is not completely the same at the source we have changed "src" and "dest" to "app.js" because we are using "app.js" and not the default "index.js".

Guide through vercel website.

Once we have made the repository on Github and created the vercel file, we can go to the Vercel website. As shown below what the site should look like, we gonna click on "Start Deploying".

Now we have to login and we are gonna continue with Github. Then we are gonne Authorize Vercel.

When we have Authorize Vercel we can now import our repository from Github.

Now we can configure our project and we only need to do 2 things here.

First is optional and that is the "Project Name" if you just want it to be the same as the repository name, then go to the next thing.

Second thing is the "Root Directory" this is important if you have a repository where you have several projects then you have to direct it to which one you want and also where you have the vercel file.

If you have several project and only what one of your project you can click edit and then you can direct to the right folder then continue. But if you don't just click deploy.

Now you can click deploy. It should also now be able to update every time you deploy to you Github repository. So try make a change and push it to your repository.
