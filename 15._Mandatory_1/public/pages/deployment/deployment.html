Deployment with Vercel

One way to deploy a node.js application is using Vercel and with this it's easy to setup.

The first thing we have to do is create a repository with our project on Github. If you are completely new to Vercel, the guide here will start from the beginning.

The vercel.json file.

After creating the Github repository, the first thing we need to do is create a vercel.json file. We will insert the code shown in the image below and push it to Github. Remember, if you have several projects in your repository and only want to deploy one of them, you have to insert the vercel file in that folder (where your app.js is located).

The source where we got it from is here How to deploy an Express API to Vercel - 4. Add vercel.json configuration.

The vercel.json file is not completely the same at the source we have changed "src" and "dest" to "app.js" because we are using "app.js" and not the default "index.js".

Guide through vercel website.

Once we have created the repository on Github and the Vercel file, we can navigate to the Vercel website. The screenshot below shows what the site should look like. We need to click on "Start Deploying".

Now we need to login and continue with Github. Then we'll authorize Vercel.

When we have authorize Vercel, we can now import our repository from Github.

Now we can configure our project and we only need to do 2 things here.

The first is optional, and that is the "Project Name." If you want it to be the same as the repository name, you can skip it and move to the next step.

The second thing is the "Root Directory." This is important if you have a repository with several projects. You need to specify which one you want to use and where you have the Vercel file.

If you have several projects and only want to deploy one of them, you can click on "edit" and then direct it to the right folder. Then, continue. But if you don't, just click on "deploy".

Now you can click deploy. It should also now be able to update every time you deploy to you Github repository. So try make a change and push it to your repository.
